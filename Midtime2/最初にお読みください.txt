●特徴
・シーンごとにスクリプト言語を切り替えて開発できます。開発メンバーごとに使用できる言語が違う場合に便利です。
　現時点では、C#、VisualBasic、JavaScript、Ruby、Python、Lua、LuaJIT、SmallBasic、HSP3、HSP2.6に対応しています。
　今後はC、C++、ノベル用スクリプト、Perl、PHPなどに対応予定です。
　現在調査中ですが、可能であればF#、D、Go、NScripter、KAG、AIMSのLUA、RealLive MAX、CatSystem2などのスクリプトにも対応していきたいです。
･できる限り単純化したAPIによりコマンドの習得が容易です。



●フォルダについて
Midtime2				Midtime2 ルートフォルダ。本フォルダの中身を配布することで他の方も、作品を起動することができます。
│  Midtime2.exe			Midtime Engine 2 ローダ（オープンソース版）
│					このファイルを起動することで、作品が開始されます。
│
│  最初にお読みください.txt		本文書です。
│
├─GameData				ゲームデータはすべてここに保存します。●変更する必要があるのはこのフォルダのみです●
│  ├─Config
│  │		KeyConfig.ini		キーボードのデフォルトキー割り当てファイルです。プレイヤーはこれとは別に設定が可能です。 [現在未対応]
│  │
│  ├─Extras				言語のサポートファイルが含まれています。通常は変更する必要はありません。
│  │					HSPで「#include」するモジュールや拡張プラグインのファイル(.as、.hsp)はここに置きます。
│  │
│  ├─Images				画像ファイル。 .bmp、.png、.jpgに対応。（.gifは準備中）
│  │
│  ├─Midtime
│  │		Midtime.ico		ゲームのアイコン画像です。.ico形式であれば、自由に変更できます。
│  │		Midtime.ini		Midtime2の設定です。画面解像度などを変更できます。 [現在未対応]
│  │
│  ├─Plugins				この中の各フォルダに所定の仕様に従って開発された外部作者によるファイルを置くことで、新しい機能が使用できるようになります。
│  │  ├─Binaries			以下のファイルが参照するファイルや、スクリプトから参照するファイルを置きます。（依存dll、HSP拡張プラグイン等）
│  │  ├─Functions			新しい関数への対応を行うファイルを置きます。
│  │  └─Languages			新しい言語への対応を行うファイルを置きます。
│  │
│  ├─Scripts				スクリプト（シーン）ファイル。１ファイルが１シーンとなり、シーンごとにスクリプト言語を変更できます。
│  │
│  ├─Sounds				サウンドファイル。.wav、.mp3のみ対応。（mp3の特許は期限が切れたようです）
│  │
│  └─Videos				動画ファイル。MPEG1のみ対応。 [現在未対応]
├─MidtimeEngine			Midtime Engine本体。フォルダ内を変更しないでください。
│  ├─Binaries
│  └─Runtimes			「Midtime2.exe」使用時、起動に必要なファイルがない場合はここのファイルを使用してお知らせが表示されます。
│
└─SaveData				プレイヤーのセーブデータ保存フォルダ。配布時は空にしてください。



●変更不可の仕様
Midtime 2では以下の挙動は仕様となっております。
主に外部ライブラリや、優先される機能の副作用が原因となっており、修正される予定はありません。

・HSP 2.6 を使用したスクリプトは１度しかロード・実行できません。
→HSP2.6の対応は、旧Midtime Engine (v1)のスクリプトやHSP2.6プラグインをそのまま動作させるためのものです。
　新規スクリプトでの使用は推奨しません。（HSP3スクリプトでは本問題は発生しません。）
　旧Midtime Engineスクリプト同士のシーン移動は、Midtime2のコマンド「MCore.Goto」ではなく、従来の「mStart」をそのまま使用してください。
　旧Midtime Engineスクリプトから他のスクリプト、またはその逆の場合は「MCore.Goto」を使用してください。



●今後の予定（本項目が完了次第正規版として公開）
・マニュアルの公開
・セーブAPIの提供。現時点ではセーブデータの保存・ロードができません。
・ある程度の動作実績



●今後の予定（正式版公開以降に対応）
・Rubyにて画面描画が行えない問題の修正
・C言語等のネイティブ言語への対応
・F#への対応（要望がある場合）
・SmallBasicの内部コンパイル
・「Midtime ノベルスクリプト」の対応。ノベルゲームがかんたんに作成できるようになります。書式は従来のMidtime Engine (v1)と同じです。
・その他上記「特徴」に記載の言語
・HSP3で「m2f "関数名"」ではなく、標準命令同様に「関数名」のみで使用できるように
・gif画像への対応
・ハイレベルなAPIの提供。ゲームでよく使われる機能（メニュー、名前入力、最初のロゴ表示など）を１行～数行のコマンドで使用できるようにします。
・ネットワーク（HTTP）APIの提供。
・サーバー用意不要のネットワークAPIの提供。
・RPGの戦闘画面やマップエディタ、アクションゲームのエディタ等のビジュアルエディタの搭載。
・有償開発サポート



●動作環境
OS: Windows 10 / 11
グラフィック: DirectX 9.0c対応、シェーダーモデル 3対応のグラフィックカード。
（Intel HD Graphics でも動作します）
※Windows 8.1、Windows7、Windows XPでの動作はサポートしておりませんが、特に意図的な制限や10のみの新しい機能は使っていませんので動作はすると思います。
　需要があるようでしたら、今後正式対応も検討しております。
※メモリ、空き容量、グラフィック等は開発されるゲームによって異なります。
